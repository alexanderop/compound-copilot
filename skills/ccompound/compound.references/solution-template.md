# Solution Template

## Bug Track Sections

Use for: `build_error`, `test_failure`, `runtime_error`, `performance_issue`, `database_issue`, `security_issue`, `ui_bug`, `integration_issue`, `logic_error`

```markdown
## Problem

1-2 sentence description of the issue and its user impact.

## Symptoms

Observable symptoms and error messages:

- Symptom 1
- Symptom 2

## What Didn't Work

| Attempted fix | Why it failed |
|--------------|---------------|
| ... | ... |

## Solution

The fix that worked. Include code snippets (before/after) when useful.

## Why This Works

Root cause explanation — what was actually happening and why the solution addresses it.

## Prevention

Concrete practice, test, or guardrail to avoid recurrence:

- ...

## Related Issues

- Links to related docs, issues, or PRs
```

## Knowledge Track Sections

Use for: `best_practice`, `documentation_gap`, `workflow_issue`, `developer_experience`

```markdown
## Context

What situation, gap, or friction prompted this guidance.

## Guidance

The practice, pattern, or recommendation. Include code examples when useful.

## Why This Matters

Rationale and impact of following (or not following) this guidance.

## When to Apply

Conditions or situations where this applies.

## Examples

Concrete before/after or usage examples showing the practice in action.
```
